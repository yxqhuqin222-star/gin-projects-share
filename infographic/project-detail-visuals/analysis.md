## Main Topic
为 Gin 项目详情页生成一组统一风格的信息图，用来替换过于简陋的流程和结构配图。

## Learning Objectives
After viewing this infographic set, the viewer should understand:
1. 每个项目的核心 workflow。
2. 每个项目的关键模块和输出。
3. Gin 网站的项目展示风格更加专业、统一。

## Target Audience
- **Knowledge Level**: Intermediate
- **Context**: 浏览个人项目详情页
- **Expectations**: 快速理解项目系统结构和实现重点

## Content Type Analysis
- **Data Structure**: process flow + system map
- **Key Relationships**: 输入、处理、输出、复盘之间的关系
- **Visual Opportunities**: numbered nodes, technical panels, output strip, dark grid background

## Key Data Points (Verbatim)
- "Summary 计算"
- "Dashboard"
- "Workbook 导出"
- "episode URL"
- "transcript"
- "message schema"
- "send API"
- "Local Storage"
- "keyword search"
- "sender filter"

## Layout × Style Signals
- Content type: process/system → suggests linear-progression and structural-breakdown
- Tone: technical/product → suggests technical-schematic
- Audience: portfolio visitors → suggests clean, high-contrast, low-noise visual system
- Complexity: moderate → suggests 6-node workflow cards with output summary

## Design Instructions (from user input)
- 配图要更美观。
- 不要只是箭头指向。
- 风格需要适配网站现在的风格。

## Recommended Combinations
1. **linear-progression + technical-schematic** (Recommended): 最适合 workflow 和系统链路，并贴合当前黑底技术风格。
2. **structural-breakdown + technical-schematic**: 适合展示模块构成和 callout。
3. **bento-grid + ui-wireframe**: 适合多模块概览，但对流程感不如线性结构强。
